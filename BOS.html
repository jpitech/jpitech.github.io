Back Office Server Troubleshooting

Reboot the POS server.

WARNING! Troubleshooting steps listed below may have serious repercussions if not followed exactly! If you are not sure of what to do, contact your OTP!

Change the KVM input to PC2 with one of the following methods:

  • Press Scroll Lock twice → 2
  • Press Scroll Lock twice → wait for the menu to appear → Select input 2 and press Enter.
  • Press the button on the switch to change the input to PC2.

You should see the POS server virtual machine by default. If you see a green background with McDonald's Host Server Easy Access, click View Virtual Machines to view the POS server. Click Shutdown VM or Shutdown OS, wait for the machine to shut down, and click Start VM.

Once the POS server finishes loading all software, check the icons on the taskbar. There should be a Java icon (looks like a coffee cup), and one that says NP. Click the Java icon and verify it says Working as Primary, then hover your mouse over the NP icon and verify there are five boxes showing If there are less than five boxes or if the Java icon is not present, call your OTP!

Once the POS server is finished booting, and you have verified it is running normally, switch back to the input 1 on the KVM switch with once of the methods above, log into QsrC&I, and retry the close.

Back Office Server Troubleshooting

Reboot the POS server.

WARNING! Troubleshooting steps listed below may have serious repercussions if not followed exactly! If you are not sure of what to do, contact your OTP!

  1. In the back office, change the KVM input to PC2 with one of the following methods:
    • Press Scroll Lock twice → 2
    • Press Scroll Lock twice → wait for the menu to appear → Select input 2 and press Enter.
    • Press the button on the switch to change the input to PC2.
  2. You should see the POS server virtual machine by default.
    • If you see a green background with McDonald's Host Server Easy Access, click View Virtual Machines to view the POS server.
  3. Click Shutdown VM or Shutdown OS.
  4. If your store is on SP10, close the window and click Restart to reboot the host server.
  5. Put the store in backup production.
    1. Go to a front counter register.
    2. Ring up a water and total out the order.
    3. "Unable to communicate with the monitors" should pop up.
    4. Tap No.
    5. Tap Yes when asked to switch to backup.
    6. Watch the text on the bottom of your KVS monitors. Once Standard switches to red, the store is in backup production.
    7. NOTE: If your store is on SP10, the store may switch to backup production automatically.
  6. Click Start VM on the POS server, and wait for the software to fully load.
  7. Check the icons on the bottom of the screen. There should be an icon that looks like the pound sign (#).
  8. Hover your mouse over the # icon and verify there are five boxes showing.
  9. If everything is running correctly, switch the store to primary production.
    1. Go to any register.
    2. Go to Manager → Support.
    3. Tap Promote to Primary Production.
      • NOTE: The switch from backup to primary is not instant! Only press the button once!
    4. Watch the text at the bottom of your KVS monitors. Once Standard switches from red to white, the store will be back in primary

After rebooting the POS server, go back to the back office server by switching to PC1 with the same method you used above.
